Q : 2 ) What is Redux Thunk used for?

Ans :
Redux Thunk is a middleware for Redux that lets you return functions instead of actions. This allows for delayed actions, such as working with promises. Redux Thunk is used to manage the state of React applications.


Q : 3 ) What is Pure Component? When to use Pure Component over Component?

Ans :
Pure components only re-render when their props or state change. They do not re-render when the value of state and props has been updated with the same values.
You can use PureComponent when:
A component's output solely depends on its state and props
There are no side effects, such as network requests or interactions with the DOM
Your React component's render() function renders the same result given the same props and state


Q : 4 ) What is the second argument that can optionally be passed to setState and what is its purpose ?

Ans :
The second argument that can optionally be passed to setState is a callback function which gets called immediately after the setState is completed and the components get re-rendered.
