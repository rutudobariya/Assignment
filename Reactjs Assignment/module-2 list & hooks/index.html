Q : 1 )Explain Life cycle in Class Component and functional component with Hooks

Class Component Lifecycle:
1. Mounting Phase:
- `constructor()`: Initialize the component's state and other properties.
- `componentWillMount()`: Executed before the component is rendered. Deprecated in React 16.3.
- `render()`: Create the initial UI structure.
- `componentDidMount()`: Called after the component is rendered to the DOM. Useful for AJAX requests and setting up event listeners.
2. Updating Phase:
- `shouldComponentUpdate(nextProps, nextState)`: Decide whether the component should re-render based on changes in props and state.
- `componentWillUpdate(nextProps, nextState)`: Executed before a re-render. Deprecated in React 16.3.
- `render()`: Re-render the component.
- `componentDidUpdate(prevProps, prevState)`: Called after the component is updated in the DOM.
3. Unmounting Phase:
- `componentWillUnmount()`: Performed just before the component is removed from the DOM. Useful for cleaning up resources and event listeners.
4. Error Handling Phase:
- `componentDidCatch(error, info)`: Introduced in React 16 for error handling within the component hierarchy.

Functional Component with Hooks Lifecycle:
1. Mounting and Updating Phase:
- `useState`: Manage the component's state within the functional component.
- `useEffect`: Handle side effects, both for mounting (similar to `componentDidMount`) and updating (similar to `componentDidUpdate`).
2. Unmounting Phase:
- You can return a cleanup function from `useEffect`, which will be called when the component is about to unmount, equivalent to `componentWillUnmount`.
3.Error Handling Phase:
- Handle errors within the component using regular JavaScript error handling mechanisms like `try...catch`. There is no direct equivalent to `componentDidCatch` in functional components. Functional components with hooks simplify the component lifecycle by allowing you to manage state and side effects in a more straightforward and functional manner. The `useEffect` hook combines what was traditionally split between `componentDidMount` and `componentDidUpdate`, making it more predictable and easier to work with.
